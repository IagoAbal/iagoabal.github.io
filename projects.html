Satisfiability testing of software requirements (2012--2013)

The purpose of this research project, developed in partnership with Educed Lda, was to explore the design of a tractable declarative language to write software requirement specifications. This language consists of a number of phrases, or boilerplates, capturing common patterns in software analysis. A requirements document made of these phrases can be encoded into a logic of quantified formulas over uninterpreted functions and integer arithmetic. We use a model finder for testing the satisfiability of such documents in order to analyze their consistency, and to obtain examples that help understanding the system being constructed.

Domain-specific term rewriting to solve bit-vector arithmetic problems (2011--2012)

Bit-blasting does not scale very well, especially in the presence of operators like multiplication or division. The main reason for this is the loss of high-level algebraic structure present in the original decision problem. However, one can exploit algebraic properties concerning the domain of bit-vectors to rewrite a problem into an equivalent, but computationally less hard, problem.

Modeling and testing Darcs's patch theory kernel (2010--2011)

In this work we capture Darcs's Patch Theory kernel in relational logic and test the satisfiability of the most well-known properties of this theory. We also present an analysis of Darcs maintainability identifying some important Darcs weaknesses. Finally, we measure the coverage of an important Darcs test suite showing that there is lot of room for improvement, and we show how to improve this coverage substantially. report (to be polished)

A Damas-Milner type system extended with predicate subtypes (2010--2012)

This thesis proposes an approach to introduce contracts as types into mainstream statically-typed functional programming languages. For this purpose, we have extended a Damas-Milner type system with predicate subtypes in the spirit of PVS, and developed a proof-of-concept programming language based on this system. In this way, the concepts of pre/postcondition and type invariant are built into the type system as contract types, and integrate smoothly with type polymorphism. engineering dissertation
